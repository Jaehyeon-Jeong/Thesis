# 4-N6.1 News SVD-Dim Grid

Status: prepared for Kaggle execution

## Goal

N6 showed that headline-only news has signal, but the first `102`-dimensional
context vector collapses in some seeds. This step tests whether reducing the
TF-IDF/SVD dimension makes the `CNN + news concat` control more stable before
moving to news-conditioned FiLM.

## Fixed Setup

- Visual input: `I60/R20/ohlc_ma_vb`
- News alignment: strict `t-1`
- Headline windows: `7d`, `20d`, `60d`
- Model: `CNN + news concat`
- Seeds: `42, 43, 44, 45, 46`
- SVD grid:
  - `svd_dim=16`, final context dim `54`
  - `svd_dim=8`, final context dim `30`

## Prepared Kaggle Cell

- [kaggle_stage4_news_context_n6_svd_dim_grid_one_cell.md](../notebooks/kaggle_stage4_news_context_n6_svd_dim_grid_one_cell.md)

The one-cell runner:

1. copies the Stage 4 and Stage 2 code snapshots;
2. builds headline windows if missing;
3. fits train-only TF-IDF/SVD vectors for `svd_dim=16` and `svd_dim=8`;
4. builds prebuilt news context artifacts for five seeds;
5. runs `CNN + news concat`;
6. writes seed-level and mean/std result tables;
7. creates one compact downloadable bundle:
   `/kaggle/working/stage4_news_context_n6_svd_dim_grid_result_bundle.zip`.

## Decision Rule

Use the smallest news vector that keeps ROC-AUC/accuracy near N6 while reducing
one-sided prediction collapse. If neither `16` nor `8` is stable, do not move
directly to news FiLM; diagnose threshold/calibration or switch to a different
news representation.

## 한국어 요약

N6의 `102`차원 news context는 일부 seed에서 signal이 있었지만 seed `43`,
`45`에서 한쪽 class로 collapse했습니다. 그래서 N7 FiLM으로 바로 넘어가기
전에, SVD 차원을 `16`, `8`로 줄여 더 안정적인 news vector를 찾습니다.
